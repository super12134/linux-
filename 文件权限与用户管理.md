# Linux 文件权限与用户管理

## 一、ACL（访问控制列表）

### 概念

ACL（Access Control List）可以对单个用户或单个组设置独立的文件权限，突破传统的 `owner / group / others` 三组权限的限制。

### 实验目标

- 用户 `a` 对文件只有**读**权限
- 用户 `b` 对文件只有**写**权限

### 操作步骤

#### 1. 环境准备（root 用户操作）

```bash
sudo -i
cd /tmp
mkdir passwd && cd passwd
mkdir shadow && cd shadow
mkdir group && cd group
touch t.txt
```

#### 2. 创建测试用户

```bash
adduser a  # 只读用户
adduser b  # 只写用户
# 设置密码（否则后续 su 切换需要密码会失败）
passwd a
passwd b
```

#### 3. 设置 ACL 权限

```bash
setfacl -m u:a:r--  t.txt
setfacl -m u:b:-w-  t.txt
```

> **权限位说明**：三位顺序为 **r（读）w（写）x（执行）**。
> - `r--` = 只读
> - `-w-` = 只写
> - `rwx` = 读写执行

查看 ACL 权限：

```bash
getfacl t.txt
```

#### 4. 测试验证

测试用户 b（只写）：

```bash
su - b
echo '223' > t.txt   # 成功（有写权限）
cat t.txt            # 失败，报 Permission denied（无读权限）
exit                 # 退回 root
```

测试用户 a（只读）：

```bash
su - a
cat t.txt            # 成功（有读权限）
```

## 二、SUID（Set User ID）

### 概念

**SUID**（Set User ID）让普通用户在执行某个程序时，**临时获得该程序文件所有者的权限**。通常用 `s` 表示在文件所有者的执行权限位上。

### 实验目标

给 `/usr/bin/cat` 命令设置 SUID 权限，使得普通用户 `b` 可以读取本无权限访问的文件。

### 操作步骤

```bash
cd /tmp/passwd/shadow/group
```

#### 1. 给 cat 命令添加 SUID

```bash
chmod u+s /usr/bin/cat
```

检查权限：

```bash
ls -l /usr/bin/cat
# 输出示例：-rwsr-xr-x  （注意所有者权限位的 s）
```

#### 2. 测试

```bash
su - b
cat t.txt            # 普通用户 b 也能读取 t.txt
exit
```

#### 3. 关闭 SUID 权限

```bash
chmod u-s /usr/bin/cat
```

> **原理**：普通用户 `b` 执行 `cat` 时，因为 `cat` 文件有 SUID，进程会以文件所有者（root）的身份运行，从而能够读取 root 权限的文件。
>
> **安全提示**：SUID 有安全风险，建议只在必要情况下临时开启，使用完毕后立即关闭。

## 三、SGID（Set Group ID）

### 概念

**SGID**（Set Group ID）让普通用户在执行某个程序时，**临时获得该程序文件所属组的权限**。通常用 `s` 表示在文件所属组的执行权限位上。

### 实验目标

先设置 `t.txt` 为 `640` 权限，使同组用户可读；再给 `cat` 命令添加 SGID，验证组权限生效。

### 操作步骤

#### 1. 设置文件权限

```bash
cd /tmp/passwd/shadow/group
chmod 640 t.txt
# 640 含义：owner 可读写(6)，group 可读(4)，others 无权限(0)
```

#### 2. 测试 SGID 前

```bash
su - b
cat t.txt            # 失败，报 Permission denied（b 不在文件所属组）
exit
```

#### 3. 给 cat 命令添加 SGID

```bash
chmod g+s /usr/bin/cat
```

#### 4. 测试 SGID 后

```bash
su - b
cat t.txt            # 成功，临时获得文件所属组的读取权限
```

## 四、Sticky Bit（黏滞位）

### 概念

**Sticky Bit** 主要用于公共目录。当一个目录设置了黏滞位后，即使目录权限为 `777`（所有人可读写执行），**用户也只能删除自己创建的文件，不能删除他人的文件**。黏滞位用 `t` 表示在 others 的执行权限位上。

> 典型应用：`/tmp` 目录默认就设置了 Sticky Bit。

### 实验目标

创建一个公共文件夹，大家都能读写，但不能删除别人的文件。

### 操作步骤

#### 1. 创建公共目录

```bash
cd /tmp/passwd/shadow/group
mkdir public
```

#### 2. 设置 Sticky Bit

```bash
chmod o+t public
```

#### 3. 检查权限

```bash
ls -l public
# 输出示例：drwxrwxrwt  （注意末尾的 t）
```

> 权限末尾的 `t` 表示 Sticky Bit 已生效。
>
> 另一种设置方式：
> ```bash
> chmod 1777 public
> ```
> 其中 `1` 就是 Sticky Bit 的数字表示。

## 附录：常用命令速查

| 命令                            | 功能              | 示例                            |
| ------------------------------- | ----------------- | ------------------------------- |
| `setfacl -m u:用户名:权限 文件` | 设置用户 ACL 权限 | `setfacl -m u:a:r-- t.txt`      |
| `setfacl -m g:组名:权限 文件`   | 设置组 ACL 权限   | `setfacl -m g:group1:rw- t.txt` |
| `getfacl 文件`                  | 查看 ACL 权限     | `getfacl t.txt`                 |
| `chmod u+s 文件`                | 添加 SUID         | `chmod u+s /usr/bin/cat`        |
| `chmod g+s 文件`                | 添加 SGID         | `chmod g+s /usr/bin/cat`        |
| `chmod o+t 目录`                | 添加 Sticky Bit   | `chmod o+t public`              |
| `ls -l`                         | 查看文件详细权限  | `ls -l t.txt`                   |

## 附录：权限位速查表

| 数字 | 二进制 | 权限  | 说明           |
| ---- | ------ | ----- | -------------- |
| 0    | 000    | `---` | 无权限         |
| 1    | 001    | `--x` | 仅执行         |
| 2    | 010    | `-w-` | 仅写           |
| 3    | 011    | `-wx` | 写 + 执行      |
| 4    | 100    | `r--` | 仅读           |
| 5    | 101    | `r-x` | 读 + 执行      |
| 6    | 110    | `rw-` | 读 + 写        |
| 7    | 111    | `rwx` | 读 + 写 + 执行 |

## 附录：特殊权限位速查表

| 特殊权限   | 数字表示 | 字母表示 | 效果                         |
| ---------- | -------- | -------- | ---------------------------- |
| SUID       | 4xxx     | `u+s`    | 临时获得文件所有者的权限     |
| SGID       | 2xxx     | `g+s`    | 临时获得文件所属组的权限     |
| Sticky Bit | 1xxx     | `o+t`    | 防删除保护（仅限自己的文件） |
