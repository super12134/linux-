# Linux 网络排查工具

## 一、连通性测试

### 1.1 `ping` — 测试网络连通性

`ping` 通过发送 ICMP 回显请求包来测试目标主机是否可达，并测量往返时间（RTT）。

```bash
ping 8.8.8.8              # 测试与 8.8.8.8 的连通性
ping -c 4 8.8.8.8         # 指定发送 4 个包后停止（Linux）
ping -n 4 8.8.8.8         # 指定发送 4 个包后停止（Windows）
ping -i 2 8.8.8.8         # 每隔 2 秒发送一个包
```

常用选项：
| 选项          | 说明                          |
| ------------- | ----------------------------- |
| `-c count`    | 发送指定次数后停止（Linux）   |
| `-n count`    | 发送指定次数后停止（Windows） |
| `-i interval` | 发送间隔（秒）                |
| `-s size`     | 指定数据包大小                |
| `-W timeout`  | 超时时间（秒）                |
| `-q`          | 静默模式，只显示摘要信息      |

## 二、路由追踪

### 2.1 `traceroute`（Linux）/ `tracert`（Windows）— 路由逐跳探测

用于定位网络路径中的断链节点，显示数据包从本机到目标主机经过的每一跳路由。

```bash
# Linux
traceroute 8.8.8.8        # 追踪到 8.8.8.8 的路由路径
traceroute -n 8.8.8.8     # 不解析主机名，直接显示 IP

# Windows
tracert 8.8.8.8           # 追踪到 8.8.8.8 的路由路径
tracert -d 8.8.8.8        # 不解析主机名
```

常用选项（Linux `traceroute`）：
| 选项     | 说明                         |
| -------- | ---------------------------- |
| `-n`     | 不解析主机名（显示 IP）      |
| `-m max` | 最大跳数（默认 30）          |
| `-q n`   | 每跳发送探测包数量（默认 3） |
| `-w sec` | 等待响应超时时间             |

### 2.2 `mtr` — ping + traceroute 结合

`mtr`（My TraceRoute）将 `ping` 和 `traceroute` 功能结合，实时动态显示每一跳的丢包率和延迟。

```bash
mtr 8.8.8.8               # 实时追踪路由并显示丢包率与延迟
mtr -r 8.8.8.8            # 以报告模式输出（适合脚本）
mtr -n 8.8.8.8            # 不解析主机名
```

常用选项：
| 选项   | 说明                               |
| ------ | ---------------------------------- |
| `-r`   | 报告模式，发送一定数量包后输出结果 |
| `-c n` | 指定发送的包数量（配合 `-r` 使用） |
| `-n`   | 不解析主机名                       |
| `-i s` | 发送间隔（秒）                     |

## 三、DNS 排查

### 3.1 `nslookup` — 域名 / IP 查询

用于查询域名的 IP 地址或反向查询 IP 对应的域名，同时显示使用的 DNS 服务器。

```bash
nslookup www.baidu.com    # 查询域名对应的 IP
nslookup 8.8.8.8          # 反向查询 IP 对应的域名
nslookup www.baidu.com 8.8.8.8   # 指定 DNS 服务器查询
```

### 3.2 `dig` — DNS 查询（更强大）

`dig`（Domain Information Groper）比 `nslookup` 更灵活、输出信息更丰富。

```bash
dig www.baidu.com         # 查询域名的 A 记录
dig @8.8.8.8 www.baidu.com  # 指定 DNS 服务器查询
dig -x 8.8.8.8            # 反向查询 IP 对应的域名
dig www.baidu.com +short  # 精简输出，只显示 IP
```

常用选项：
| 选项      | 说明                      |
| --------- | ------------------------- |
| `@server` | 指定 DNS 服务器           |
| `-x ip`   | 反向查询                  |
| `+short`  | 只显示精简结果            |
| `+trace`  | 从根 DNS 开始逐级追踪解析 |
| `ANY`     | 查询所有记录类型          |

### 3.3 `host` — 简易 DNS 查询

```bash
host www.baidu.com        # 查询域名 IP
host 8.8.8.8              # 反向查询
```

## 四、端口与服务诊断

### 4.1 `ss` — 查看本机端口监听和连接（推荐）

`ss`（Socket Statistics）是 `netstat` 的现代替代方案，性能更好、输出更快。

```bash
ss -lntp                  # 查看所有监听的 TCP 端口及进程
ss -antp                  # 查看所有 TCP 连接
ss -anup                  # 查看所有 UDP 连接
ss -tuln                  # 查看所有监听的 TCP + UDP 端口
ss -s                     # 查看网络统计信息
```

常用选项：
| 选项 | 说明                             |
| ---- | -------------------------------- |
| `-l` | 只显示监听中的端口               |
| `-n` | 不解析服务名（显示端口号）       |
| `-t` | 只显示 TCP                       |
| `-u` | 只显示 UDP                       |
| `-p` | 显示占用端口的进程 PID 和名称    |
| `-a` | 显示所有连接（包括监听和非监听） |
| `-s` | 显示网络统计摘要                 |

### 4.2 `netstat` — 传统网络连接查看

```bash
netstat -an               # 查看所有连接
netstat -an | grep :22    # 查看指定端口（如 22）
netstat -lntp             # 查看监听的 TCP 端口（需 root）
netstat -r                # 查看路由表
```

常用选项：
| 选项 | 说明                           |
| ---- | ------------------------------ |
| `-a` | 显示所有连接和监听端口         |
| `-n` | 不解析名称（显示 IP 和端口号） |
| `-t` | 显示 TCP                       |
| `-u` | 显示 UDP                       |
| `-l` | 只显示监听中的端口             |
| `-p` | 显示进程 PID 和名称            |
| `-r` | 显示路由表                     |

### 4.3 `telnet` — 测试端口可达性

```bash
telnet 8.8.8.8 53         # 测试目标 IP 的 53 端口是否开放
telnet baidu.com 80       # 测试域名 80 端口
```

### 4.4 `nc`（Netcat）— 网络测试瑞士军刀

```bash
nc -zv 8.8.8.8 53         # 扫描指定端口是否开放
nc -zv 8.8.8.8 1-1000    # 扫描端口范围
nc -l -p 8888             # 在本机 8888 端口开启监听
```

常用选项：
| 选项      | 说明                   |
| --------- | ---------------------- |
| `-z`      | 只扫描端口，不发送数据 |
| `-v`      | 详细输出               |
| `-l`      | 监听模式               |
| `-p port` | 指定端口               |
| `-n`      | 不解析 DNS             |

### 4.5 `lsof` — 查看进程打开的文件和端口

```bash
lsof -i :80               # 查看 80 端口被哪个进程占用
lsof -nP -iTCP -sTCP:LISTEN  # 查看所有监听的 TCP 端口
lsof -i UDP               # 查看所有 UDP 连接
```

常用选项：
| 选项       | 说明                  |
| ---------- | --------------------- |
| `-i`       | 显示网络连接          |
| `-i :port` | 查看指定端口的进程    |
| `-n`       | 不解析主机名          |
| `-P`       | 不解析端口名          |
| `-u user`  | 查看某用户的进程      |
| `-p PID`   | 查看某 PID 打开的文件 |

## 五、HTTP 请求测试

### 5.1 `curl` — 发送 HTTP 请求

```bash
curl http://www.baidu.com           # 发送 GET 请求并返回响应内容
curl -I http://www.baidu.com        # 只查看响应头（Header）
curl -v http://www.baidu.com        # 显示详细请求和响应信息
curl -o file http://www.baidu.com   # 将响应内容保存到文件
curl -X POST -d "key=value" http://example.com  # 发送 POST 请求
```

常用选项：
| 选项        | 说明                            |
| ----------- | ------------------------------- |
| `-I`        | 只获取响应头（HEAD 请求）       |
| `-v`        | 显示详细请求和响应信息          |
| `-o file`   | 将输出写入文件                  |
| `-L`        | 跟随重定向                      |
| `-X METHOD` | 指定请求方法（GET/POST/PUT 等） |
| `-d data`   | 发送 POST 数据                  |
| `-H header` | 添加自定义请求头                |

### 5.2 `wget` — 下载文件 / 测试 URL

```bash
wget http://www.baidu.com           # 下载网页
wget -O output.html http://example.com  # 指定输出文件名
wget --spider http://example.com    # 只检查 URL 是否存在，不下载
```

常用选项：
| 选项       | 说明                       |
| ---------- | -------------------------- |
| `-O file`  | 指定输出文件名             |
| `--spider` | 只检查链接是否有效，不下载 |
| `-q`       | 静默模式                   |
| `-P dir`   | 指定下载目录               |

## 六、抓包分析

### 6.1 `tcpdump` — 命令行抓包

`tcpdump` 是 Linux 下最强大的命令行抓包工具，用于捕获和分析网络数据包。

```bash
tcpdump -i eth0             # 抓取 eth0 网卡的所有流量
tcpdump -i eth0 -nn         # 不解析主机名和端口名
tcpdump host 8.8.8.8        # 抓取与 8.8.8.8 通信的包
tcpdump port 80             # 抓取 80 端口的流量
tcpdump -i eth0 -w dump.pcap   # 抓包并保存到文件
tcpdump -r dump.pcap        # 读取并分析抓包文件
tcpdump -i eth0 tcp and port 443  # 抓取 HTTPS 流量
```

常用选项：
| 选项           | 说明                          |
| -------------- | ----------------------------- |
| `-i interface` | 指定抓包网卡                  |
| `-nn`          | 不解析主机名和端口名          |
| `-w file`      | 将抓包结果写入文件            |
| `-r file`      | 读取抓包文件                  |
| `-c count`     | 抓取指定数量包后停止          |
| `-X`           | 以十六进制和 ASCII 显示包内容 |
| `-v` / `-vv`   | 显示更详细的包信息            |

过滤表达式示例：
```bash
tcpdump src 192.168.1.1            # 只抓取源 IP 为指定地址的包
tcpdump dst 8.8.8.8                # 只抓取目标 IP 为指定地址的包
tcpdump port not 22                # 排除 SSH（22 端口）流量
tcpdump tcp and port 80            # TCP 且端口为 80
tcpdump udp or port 53             # UDP 包或 DNS（53 端口）
```
