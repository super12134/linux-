# Linux 进程管理

## 一、进程简介

**进程（Process）** 是程序的一次执行过程，是系统进行资源分配和调度的基本单位。

- **程序**：静态的代码文件
- **进程**：动态的执行实体，拥有独立的地址空间、资源句柄等

**进程状态**：
| 状态           | 说明                               |
| -------------- | ---------------------------------- |
| R (Running)    | 正在运行或在运行队列中             |
| S (Sleeping)   | 休眠，等待某事件完成               |
| D (Disk Sleep) | 不可中断休眠，通常等待 I/O         |
| T (Stopped)    | 暂停 / 停止                        |
| Z (Zombie)     | 僵尸进程，进程已结束但父进程未回收 |
| X (Dead)       | 已死亡（几乎看不到）               |

## 二、进程查看命令

### 2.1 `ps` — 静态进程快照

```bash
ps               # 查看当前终端下的进程
ps -ef           # 查看所有进程详细信息
ps aux           # 以 BSD 风格查看所有进程
```

常用字段说明：
- `PID`：进程 ID
- `PPID`：父进程 ID
- `%CPU` / `%MEM`：CPU / 内存占用百分比
- `STAT`：进程状态
- `COMMAND`：执行的命令

### 2.2 `top` — 动态进程监控（实时刷新）

```bash
top
```

`top` 进入交互界面后的快捷键：

| 快捷键 | 功能                          |
| ------ | ----------------------------- |
| `P`    | 按 CPU 使用率降序排列         |
| `M`    | 按内存使用率降序排列          |
| `T`    | 按 CPU 累计占用时间排列       |
| `q`    | 退出 top                      |
| `1`    | 查看多核 CPU 各个核的使用情况 |
| `c`    | 查看进程的完整命令行路径      |
| `N`    | 按 PID 号排列                 |
| `空格` | 立即刷新                      |

### 2.3 `htop` — 增强版 top（需安装）

```bash
htop            # 更直观的交互界面，支持鼠标操作
```

### 2.4 `pstree` — 进程树

```bash
pstree           # 以树形结构显示进程父子关系
pstree -p        # 同时显示 PID
```

## 三、进程跟踪 / 调试命令

### 3.1 `strace` — 跟踪系统调用和信号

`strace` 用于追踪进程发出的系统调用和接收到的信号，是调试和分析程序的利器。

```bash
# 跟踪已运行的进程
strace -p <PID>

# 跟踪执行命令时的系统调用
strace cat jupyter.txt
```

常用选项：
| 选项                 | 说明                   |
| -------------------- | ---------------------- |
| `-p PID`             | 跟踪指定 PID 的进程    |
| `-e trace=open,read` | 只跟踪指定系统调用     |
| `-o file`            | 将输出写入文件         |
| `-t`                 | 打印时间戳             |
| `-c`                 | 统计系统调用次数和时间 |

## 四、文件 / 端口诊断

### 4.1 `lsof` — 查看进程打开的文件和端口

`lsof`（List Open Files）用于列出当前系统打开的文件、目录、网络连接等。

```bash
# 查看指定端口占用情况
lsof -i :22

# 监听本机所有 TCP 端口（不解析主机名/端口名）
lsof -nP -iTCP -sTCP:LISTEN
```

常用选项：
| 选项       | 说明                       |
| ---------- | -------------------------- |
| `-i`       | 显示网络连接               |
| `-i :port` | 查看指定端口的进程         |
| `-n`       | 不解析主机名（显示 IP）    |
| `-P`       | 不解析端口名（显示端口号） |
| `-u user`  | 查看某用户的进程           |
| `-p PID`   | 查看某 PID 打开的文件      |

## 五、进程控制命令

### 5.1 `kill` — 终止进程

```bash
kill <PID>                 # 默认发送 SIGTERM（15），请求正常终止
kill -9 <PID>              # 发送 SIGKILL（9），强制终止
kill -l                    # 列出所有信号类型
```

常用信号：
| 信号    | 编号 | 说明                   |
| ------- | ---- | ---------------------- |
| SIGTERM | 15   | 请求终止（默认）       |
| SIGKILL | 9    | 强制杀死               |
| SIGSTOP | 19   | 暂停进程               |
| SIGCONT | 18   | 继续被暂停的进程       |
| SIGHUP  | 1    | 挂起（常用于重载配置） |

### 5.2 `killall` — 按名称终止进程

```bash
killall nginx              # 杀死所有名为 nginx 的进程
killall -9 firefox         # 强制杀死所有 firefox 进程
```

### 5.3 `pgrep` / `pkill` — 基于模式匹配

```bash
pgrep -u root              # 查找 root 用户的进程 PID
pkill -u root sshd         # 杀死 root 用户的 sshd 进程
```

## 六、前台 / 后台进程管理

### 6.1 后台运行 &

```bash
sleep 100 &                # 在后台运行命令
```

### 6.2 `jobs` — 查看后台任务

```bash
jobs                       # 列出当前终端的后台任务
jobs -l                    # 同时显示 PID
```

输出示例：
```
[1]  Running                 sleep 100 &
[2]- Running                 sleep 200 &
[3]+ Running                 sleep 300 &
```

- `+` 表示当前默认任务
- `-` 表示上一个默认任务

### 6.3 `bg` — 将任务放到后台继续运行

```bash
bg %1                      # 将编号为 1 的任务放到后台运行
```

### 6.4 `fg` — 将后台任务拉回前台

```bash
fg %1                      # 将编号为 1 的任务拉到前台
fg                         # 将最近的默认任务拉到前台
```

## 七、进程持久化运行

### 7.1 `nohup` — 终端关闭后继续运行

```bash
nohup command &             # 终端退出后进程仍在后台运行
nohup python script.py > output.log 2>&1 &
```

- 标准输出默认重定向到 `nohup.out`
- `2>&1` 将错误输出也重定向到标准输出

### 7.2 `screen` / `tmux` — 终端复用器（推荐）

```bash
screen -S mysession         # 创建新会话
# Ctrl+A D                  # 分离会话（detach）
screen -r mysession         # 重新连接会话
```

```bash
tmux new -s mysession       # 创建新会话
# Ctrl+B D                  # 分离
tmux attach -t mysession    # 重新连接
```

## 八、守护进程简介

**守护进程（Daemon）** 是长期运行在后台、不受终端控制的服务进程，名称通常以 `d` 结尾，例如：
- `sshd` — SSH 服务
- `nginx` — Web 服务器
- `mysqld` — MySQL 数据库
- `systemd` — 系统和服务管理器

```bash
systemctl status sshd       # 查看 sshd 守护进程状态
systemctl start nginx       # 启动 nginx 守护进程
systemctl enable docker     # 设置 docker 开机自启
```
